Deploy and Launch the FlightTracker3DApp

------------------------------------------------------------------

1)
Deploying FlightTracker3DApp on Raspberry Pi:

Build the Application in RELASE mode.

Select FlightTracker3DApp as startup project.

IMPORTANT ! !
In Solution Explorer select the FlightTracker3DApp project 

Build --> Publish --> Folder:
	Target folder: ..\publishFlightTracker3DApp\

	Profile settings 
		Deployment mode: Self contained
		Runtime: linux-arm64



Build --> Publish --> Click "Publish" button


Here you find concise steps to enable passwordless SSH from your Windows machine to the Raspberry Pi (recommended: use ed25519 keys):
	raspberry_pi_best_practice_start_guide.txt

Copy to Raspberry Pi:

/*  C:\Users\niels\source\repos\FlightTracker3D\publishFlightTracker3DApp   */


Open a powershell

Navigate to the folder where the application is built.
C:\Users\niels\source\repos\FlightTracker3D

Open a terminal and connect to the Raspberry Pi via SSH:
ssh pi@raspberrypi

Create a folder for the application:
mkdir FlightTracker3D

From windows PC:
scp -r publishFlightTracker3DApp\* pi@raspberrypi:~/FlightTracker3D

Of unknown reason, the .json file is not copied with the above command. So copy it separately:
scp -r .\publishFlightTracker3DApp\FlightTracker3DAppSettings.json pi@raspberrypi:~/FlightTracker3D

From Raspberry Pi:

sudo raspi-config
Enable: Interface Options --> I2C --> Yes

reboot

cd FlightTracker3D/
chmod +x ./FlightTracker3DApp

2) 
Setup the Application Settings: 
cd FlightTracker3D/
nano ./FlightTracker3DAppSettings.json
-->
{
  "FlightTracker3DAppSettings": {
    "MinimumLogLevel": "Warning",
    "HardwareModes": {
      "LcdHardwareMode": "Emulated",
      "LedHardwareMode": "Emulated",
      "StepperMotorHardwareMode": "Emulated"
    }
  }
}

------------------------------------------------------------------

3)
Setup Readsb (ADS-B Decoder):

Readsb should already be installed and located in /home/pi/readsb/
If not installed, follow the readsb installation instructions.

Verify readsb is working manually (optional):
cd ~/readsb
./readsb --device-type rtlsdr --gain auto --interactive --net --net-sbs-port=30003

Press Ctrl-C to stop the manual test.

------------------------------------------------------------------

4)
Setup SystemD Services for Auto-Start at Boot:

The system will automatically:
1. Start Readsb service first
2. Wait 4 seconds for Readsb to initialize
3. Start FlightTracker3D service

Copy service files to Raspberry Pi:
From Windows PC:
scp flighttracker3d.service pi@raspberrypi:~/FlightTracker3D/
scp readsb.service pi@raspberrypi:~/FlightTracker3D/
scp setup_service.sh pi@raspberrypi:~/FlightTracker3D/

From Raspberry Pi:
cd FlightTracker3D/
chmod +x setup_service.sh
./setup_service.sh

The setup script will:
- Add user 'pi' to gpio, i2c, and spi groups
- Install Readsb systemd service
- Install FlightTracker3D systemd service (with dependency on Readsb)
- Enable auto-start at boot for both services
- Start both services (Readsb first, then FlightTracker3D after 4 seconds)

Useful service commands:

Readsb:
sudo systemctl status readsb.service               # Check status
sudo journalctl -u readsb.service -f               # View logs in real-time
sudo systemctl stop readsb.service                 # Stop service
sudo systemctl start readsb.service                # Start service
sudo systemctl restart readsb.service              # Restart service

FlightTracker3D:
sudo systemctl status flighttracker3d.service      # Check status
sudo journalctl -u flighttracker3d.service -f      # View logs in real-time
sudo systemctl stop flighttracker3d.service        # Stop service
sudo systemctl start flighttracker3d.service       # Start service
sudo systemctl restart flighttracker3d.service     # Restart service

Both services:
sudo systemctl stop flighttracker3d.service readsb.service     # Stop both
sudo systemctl start readsb.service                            # Start both (FT3D auto-starts)
sudo systemctl disable flighttracker3d.service readsb.service  # Disable auto-start

Note: After running setup_service.sh, reboot for group changes to take effect:
sudo reboot

After reboot, both services will start automatically with correct timing.

------------------------------------------------------------------

5)
Launching the Application Manually (without service):

Open a terminal and connect to the Raspberry Pi via SSH:
ssh pi@raspberrypi

From Raspberry Pi:
cd FlightTracker3D/
./FlightTracker3DApp

Press Ctrl-X to shutdown gracefully (stepper motors will return to home position).

------------------------------------------------------------------Launching the Application:

Open a terminal and connect to the Raspberry Pi via SSH:
ssh pi@raspberrypi

From Raspberry Pi:
cd FlightTracker3D/
./FlightTracker3DApp
